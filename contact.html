Contact

Feedback is part of the product process.

Legacyless is still early, and careful feedback matters. If you try the beta on real C++ code, the most helpful thing you can do is share what felt solid, what felt unclear, and what should improve before the ruleset grows further.

Best feedback

Useful reports include

  • The rule you ran
  • The preview or diff you expected
  • The output Legacyless produced
  • Enough project context to reproduce the behavior

Primary beta channel

Direct beta email

Use a direct mailbox for feedback that includes project context, screenshots, or examples you do not want to post publicly.

beta@legacyless.dev

Public discussion

GitHub issues or discussions

Keep public bug reports and product suggestions in a shared place so patterns are easier to spot as the beta develops.

Beta builds and notes

Pair feedback with the exact beta build you tested so behavior can be traced back to the release you used.

What to send

Focus on specifics over volume.

  • Share one concrete rule outcome at a time when possible.
  • Include whether the result was correct, surprising, or incomplete.
  • Mention if the CLI workflow or VS Code extension experience felt awkward.
  • Call out anything that affects trust, not just correctness.
